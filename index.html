02 — Solution Approach

AI-Powered Signal Traceability

Four capabilities that together make the invisible signal chain visible, queryable, and safe to refactor.

01 / 04 · Parsing

Multi-Layer Signal Parsing

Five purpose-built parsers extract structured data from every layer of the Parallax signal chain — no generated code or API calls required. All parsers are orchestrated by a single scanner entry point.

L1
RvmParser — Vehicle Model
Reads *.rvm YAML files; extracts properties and services; derives NATS subjects algorithmically (no codegen)
L2
RoutingRulesParser — EGW2 Bridge
Maps Elpis ↔ Parallax namespace from routing-rules/*.yaml; classifies direction per route
L3
RalBusParser — CAN/ETH Signals
Captures PDU IDs, sender ECU, receiver ECUs, bus type and cycle time from *.bus.ral
L4
AppCodeParser — Kotlin Apps
Grep-based analysis finds buildFlow(), requestTyped() and handleTypedRequests() call sites
L5
OrchestratorParser — Fan-out
Resolves fan-out recipes: one incoming service call expands to N downstream EGW2 signal edges

02 / 04 · Graph

Signal Flow Graph — 3 Confirmed Patterns

Parsed data populates a directed signal graph. Edges encode three flow patterns confirmed from the starter codebase: command, state, and fan-out. The graph powers BFS blast-radius and impact analysis.

A
Pattern A — Command (App → Zonal)
App publishes SERVICE.VSVC.* → EGW2 translates → Zonal ECU CAN frame (e.g. setFanSpeed → 0x700)
B
Pattern B — State (Zonal → App)
Zonal ECU sends CAN frame → EGW2 translates → App subscribes STATE.VEHICLE.* (e.g. fan_speed readout)
C
Pattern C — Fan-out (Orchestrator)
App calls Orchestrator → recipe expands to N parallel EGW2 service calls (e.g. setDriveMode → 6 ECU signals)
DB
parallax_flow_edges — with confidence scores
Graph stored in PostgreSQL; BFS traversal enables blast-radius and variant-diff queries via Cytoscape.js

03 / 04 · AI Skills

AI Skills + MCP Tool Integration

KB AI skills wrap the signal graph with LLM-powered analysis, answering questions no SQL query can answer alone. Results surface via REST API, Cytoscape graph UI, and a callable MCP tool.

01
Skill Collector — Evidence Gathering
Pre-fetches signal graph context from PostgreSQL before LLM prompt assembly; scoped per-domain or per-variant
02
LLM Analysis — Trace & Impact Skills
Signal trace, blast-radius, variant diff, and pattern detection skills defined via SKILL.md contracts and schemas
03
Skill Populator — Validate & Persist
Validates LLM output against schema; writes to structured_views; surfaces via 4 tab sections in the UI
04
MCP Tool — parallax_trace(signal)
Callable from IDE, CI pipeline, or chat: returns full chain with Git SHA, confidence score, affected apps, and blast radius
